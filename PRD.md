# 📄 개인용 유튜브·애드센스 자산 통합 관리 시스템 PRD V0.1.6

## 1. 프로젝트 개요
* **목적:** 다수의 유튜브 채널, 애드센스 계정, 휴대폰 및 구매, 수입/지출 내역을 한곳에서 통합 관리하고 보안을 유지하는 1인용 통합 관리 웹 어플리케이션.
* **타겟 사용자:** 관리자 1인 (독점 사용)
* **핵심 가치:** 강력한 필드 데이터 암호화, 직관적인 UI, 데이터 유기적 연동 및 수입/지출 시각화.
* **개발 기조:** 초보 1인 개발 및 AI 코딩 어시스턴트 활용에 최적화되도록, **복잡한 보안/상태 관리는 반드시 검증된 표준 라이브러리를 사용하여 보일러플레이트 코드를 최소화**한다.

---

## 2. 권장 기술 스택 (Tech Stack & Libraries)
* **프레임워크:** Next.js App Router (Vercel 배포)
* **데이터베이스:** MongoDB + Mongoose (MongoDB Atlas)
* **[핵심] 인증 및 세션:** **NextAuth.js (Auth.js)** - Credentials Provider 사용
* **[핵심] 데이터 암호화:** **mongoose-field-encryption** (Mongoose 플러그인 활용)
* **전역 상태 관리:** Zustand
* **UI/UX 라이브러리:** Tailwind CSS, dnd-kit (드래그 앤 드롭)
* **문서 내보내기:** `html2canvas` + `jspdf` (클라이언트 사이드 렌더링)

## 3. 핵심(글로벌) 및 보안 요구사항
* **표준 라이브러리 기반 인증 (NextAuth):** 마스터 비밀번호 인증 및 API 라우트 보호(JWT 세션 검증)는 직접 구현하지 않고 NextAuth.js의 아키텍처에 전적으로 위임합니다.
* **플러그인 기반 필드 암호화:** 비밀번호, API Key, 계좌번호 등 민감한 내용은 DB 저장 전 수동으로 암호화하지 않고, `mongoose-field-encryption` 플러그인을 스키마에 적용하여 저장 시 자동 암호화, 조회 시 자동 복호화되도록 처리합니다. (단, 검색 연동이 필요한 이메일 등은 별도의 SHA-256 해시 필드 `_hash`를 추가로 두어 검색에 활용)
* **환경 변수 격리 (No Hardcoding):** MongoDB URI, NextAuth Secret, 암호화 키 등은 절대 코드에 포함하지 않고 Vercel의 `Environment Variables`에 등록하여 관리합니다.
* **비상 복구 키 시스템:** 시스템 최초 설정 시 24자리 무작위 '비상 복구 키'를 발급하여, 비밀번호 분실 시 재설정할 수 있는 백도어를 제공합니다.
* **글로벌 정보 보호 (Global Blur):** Zustand를 활용해 전역 상태(`isBlurred`)를 관리하며, 클릭 한 번으로 모든 화면의 민감 정보를 블러 처리합니다.

## 4. 상세 기능 명세 (UI 탭 기준)

### [탭 1] 유튜브 계정 
* **글로벌 보안 블러 버튼:** 화면 상단 배치 (상태 토글).
* **입력 데이터:** 계정ID(이메일), 비밀번호, 애드센스 계정, 전화번호, 생성/구매 여부, 사용 유무, API Key, 애드센스 상태, OTP 3종.
* **조건부 입력:** '생성' ➔ [생성 날짜] / '구매' ➔ [구매처, 판매자, 가격(USD), 구매일, 생성일].
* **자동 환율 변환:** USD 입력 시 KRW 자동 변환 (API 또는 수동 롤백).
* **UI/UX (DnD):** 드래그로 행 이동. 잦은 DB 통신을 막기 위해 프론트(Zustand)에서 순서 유지 후 **'순서 저장' 버튼 클릭 시 Bulk Update**.

### [탭 2] 애드센스 및 휴대폰 
* **입력 데이터:** 계정ID, 비밀번호, 명의자, 은행, 계좌번호, 전화번호, 주소, 신청/도착일, 계정 상태, 휴대폰 모델, 구매처, 가격(KRW), OTP 3종, 구매일.
* **OTP 데이터 자동 연동:** 입력한 계정ID의 해시값이 [탭 1]과 일치하면, [탭 1]의 비밀번호 및 OTP 데이터를 즉시 불러와 연동 (Mongoose 쿼리).

### [탭 3] 채널 관리
* **입력 데이터:** 채널명, 핸들명, 카테고리, 국가, 생성일, 수입 발생 여부, 상태, 연결된 계정(유튜브/애드센스), 수창 구분.
* **관계 연동:** 연결된 계정은 Mongoose `ObjectId` (Ref) 방식으로 선택.
* **수창 조건부 입력:** 자력 수창 ➔ [수창 날짜] / 구매 ➔ [구매처, 판매자, 가격(USD), 구매일, 시작일, 국가, 카테고리].

### [탭 4] 수입 및 지출 관리
* **자동화 (Mongoose Hook 활용):** 탭 1, 2, 3에서 구매 데이터(가격)가 생성/수정될 때 Mongoose `pre-save` 또는 `post-save` 훅을 활용하여 지출 컬렉션에 사유와 함께 자동 적재합니다.
* **통계 내역:** 예상 수입 합산, 지출 필터링, 최종 순이익 및 기간별 시각화.

### [탭 5] 통합 대시보드 및 리포트
* **경고 관리 (탭 5):** 탭 1~3의 '경고/삭제' 상태 데이터를 Mongoose 쿼리로 모아 발생 사유 기록.
* **채널 조회 (탭 5 - Populate 활용):** 비정규화(데이터 중복 저장)를 피하고, Mongoose의 `.populate()`를 사용하여 백엔드에서 채널+유튜브계정+애드센스계정 데이터를 실시간 조인하여 프론트에 전달.
* **클라이언트 내보내기:** 브라우저 자원(`html2canvas`)을 이용해 PDF 및 Markdown 내보내기 수행 (Vercel 서버리스 타임아웃 방지).

## 6. AI 코딩 어시스턴트(Cursor/Claude) 활용 지침 (Meta Prompt)
> **💡 시스템 프롬프트 가이드:** 이 프로젝트를 보조하는 AI는 다음 원칙을 반드시 준수하여 코드를 생성해야 합니다.
1. **Never reinvent the wheel:** 인증 및 세션 코드를 요청받으면 무조건 `NextAuth.js` 코드를 작성하라. 암호화를 요청받으면 수동 `crypto` 로직 대신 `mongoose-field-encryption` 플러그인 세팅을 제시하라.
2. **Use Server/Client strictly:** UI 조작(dnd-kit, onClick), 상태 관리(Zustand)가 들어간 컴포넌트는 반드시 최상단에 `'use client';`를 명시하라. DB 연결 및 모델 로직은 Server 측에서만 동작하도록 분리하라.
3. **Step-by-Step execution:** 사용자가 전체 탭을 한 번에 만들어달라고 요청하더라도, 컴포넌트를 잘게 쪼개어(예: Schema 정의 먼저 -> API 라우트 -> UI 컴포넌트) 단계별로 제시하라.
